Framexsを勉強する上で、助けとなるサービスのご紹介

　例：芸能人の阿部寛（あべひろし）のウェブサイトのように、動的生成要素の無いHTMLを使って作成したウェブサイトのこと。（業界では、けっこう有名なサイト例です）
http://abehiroshi.la.coocan.jp/
　WordPressなど、動的にウェブサイトデータを生成するCMSなどと違い、別途データベースが必要ない。また、サーバー側のスペックも動的サイト運用より静的サイト運用の方が低くて済む。

　データを受け取るクライアントのスマホやPCのスペックが低かったり、インターネット回線速度が若干遅くても、ウェブサイトの表示が比較して非常に早く、SEO上の大きなメリットがある。

　静的コンテンツのみがウェブサーバーに置かれる為、セキュリティーホールが少なくなるので、サイトを維持する為のメンテナンスコストも低く済む。
　Framexsは、静的ページを作る為の他のソフトウェアでは面倒な、修正箇所を一括してサイトへ適用することのできるテンプレート化に優れています。

　たとえば、ヘッダー部分のHTMLに何か追記したいhtml文章やスクリプトがあった場合、他の静的ページを作るソフトでは一枚一枚、全ページに対して書き換え作業が必要になる場合もあります。

　しかし、Framexsでは、実に少ない作業で全ページに対して修正箇所を適用することが可能なため、書き換えの為の単純作業を劇的に減らす事ができます。

　Framexsでは静的なページを作るためのフレームワークとして働き、プログラマー（たとえばJavaScriptなど）とマークアップエンジニアの分業を進める事ができる。
　静的サイトを作る為の有利な機能性を知った今、Framexsを試してみようという気になったでしょうか。
　Framexsはまだ日本国内において知名度が低いですが、活用すれば素晴らしいワークフローをあなたのウェブ制作に適用できます。
